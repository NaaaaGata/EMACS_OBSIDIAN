# EMACS_OBSIDIAN 日本語ユーザーガイド

このガイドでは、EMACS_OBSIDIANのインストールから日常的なノート編集、Wikiリンク、LaTeX数式、グラフビュー、設定変更、トラブル対処までを説明します。

## 1. EMACS_OBSIDIANとは

EMACS_OBSIDIANは、MarkdownファイルをObsidianのように相互リンクしながらEmacsで編集するためのパッケージです。

`M-x obsidian`を実行すると、画面が次の3つに分かれます。

```text
┌──────────────┬──────────────────────────┬──────────────────┐
│ ファイル階層 │ Markdown編集画面         │ リンクグラフ     │
│              │                          │                  │
│ 左           │ 中央                     │ 右               │
└──────────────┴──────────────────────────┴──────────────────┘
```

- 左：Vault内のフォルダとMarkdownファイルを表示します。
- 中央：選択したMarkdownファイルを編集します。
- 右：同じフォルダにあるノート間のリンク関係を表示します。

ここでいう「Vault」は、ノートを保存する基準フォルダのことです。通常のフォルダなので、ほかのエディタやファイル管理ソフトからも内容を扱えます。

## 2. 必要な環境

- Emacs 27.1以上
- Markdownノートを保存するフォルダ
- 画像による数式プレビューを使う場合は`latex`と`dvipng`

`latex`や`dvipng`がなくても、ノート編集、リンク、グラフなどの機能は利用できます。

## 3. インストール

### 3.1 リポジトリを取得する

ターミナルで次を実行します。

```sh
git clone https://github.com/NaaaaGata/EMACS_OBSIDIAN.git
```

### 3.2 init.elに登録する

Emacsの`init.el`に次を追加します。パスは、実際にリポジトリを配置した場所へ変更してください。

```elisp
(add-to-list 'load-path "~/path/to/EMACS_OBSIDIAN")
(require 'obsidian)
```

毎回同じVaultを開く場合は、次の設定も追加します。

```elisp
(setq obsidian-vault-directory "~/Documents/MyVault")
```

まだVaultが存在しない場合は、先にフォルダを作成します。

```sh
mkdir -p ~/Documents/MyVault
```

### 3.3 設定を反映する

Emacsを再起動するか、`init.el`の設定を評価します。その後、次を実行します。

```text
M-x obsidian RET
```

`obsidian-vault-directory`を設定していない場合は、ミニバッファでVaultの場所を尋ねられます。

## 4. まずデモを試す

リポジトリには動作確認用Vaultが付属しています。

1. `M-x obsidian`を実行します。
2. Vaultとして`EMACS_OBSIDIAN/examples/demo-vault`を指定します。
3. 左パネルの`music/`をマウスでクリックするか、カーソルを合わせて`RET`を押します。
4. `music.md`を開きます。

右側には`music.md`と`guitar.md`を中心に、楽器ノートが接続されたグラフが表示されます。

## 5. 3つのパネルを移動する

Emacs標準のウィンドウ移動操作を利用できます。

| 操作 | 内容 |
|---|---|
| `C-x o` | 次のパネルへ移動 |
| マウスクリック | クリックしたパネルへ移動 |

起動直後は左のファイル階層にカーソルがあります。

## 6. 左パネル：ファイル階層

左パネルではVault内のフォルダと`.md`ファイルを操作します。

| 操作 | 内容 |
|---|---|
| マウス左クリック | フォルダを開閉、またはノートを開く |
| `RET` | カーソル位置のフォルダを開閉、またはノートを開く |
| `TAB` / `→` | フォルダを展開 |
| `←` | フォルダを折りたたむ |
| `n` | 新しいノートを作成 |
| `g` / `r` | 表示を更新 |
| `<` / `>` | 左パネルの幅を変更 |

フォルダを選ぶと、そのフォルダがグラフの表示範囲になります。たとえば`music`フォルダと`sns_idea`フォルダがある場合、`music`を選択している間は`sns_idea`のノートはグラフに混ざりません。

## 7. ノートを作成する

左パネルで`n`を押すか、編集画面で`C-c o n`を押します。通常のノート名を入力すると、現在選択しているフォルダ内に`.md`ファイルが作成され、グラフも同じフォルダの表示を維持します。

サブフォルダ内に作る場合は、名前にパスを含めます。

```text
music/drums
```

この場合は`music/drums.md`が作成されます。新しいノートには見出しと作成日時が自動挿入されます。

```markdown
# drums

[2026-07-14 Tue 14:30]
```

日時の書式は`obsidian-timestamp-format`で変更できます。

## 8. 中央パネル：Markdown編集

中央パネルは通常のEmacsバッファです。文章を編集したら`C-x C-s`で保存します。

| 操作 | 内容 |
|---|---|
| `C-x C-s` | ノートを保存 |
| `C-c o n` | 新規ノートを作成 |
| `C-c o l` | Wikiリンクを挿入 |
| `M-RET` / `C-c o f` | カーソル位置のリンクを開く |
| `C-c o b` | 直前に開いていたノートへ戻る |
| `C-c o t` | LaTeXプレビューを切り替える |
| `C-c o g` | グラフを更新 |
| `C-c o r` | ファイル階層を更新 |

## 9. Wikiリンクを使う

### 9.1 基本形式

別のノートへリンクするには、ノート名を二重角括弧で囲みます。

```markdown
[[music]]
```

表示名を変えたい場合は`|`の後ろに表示名を書きます。

```markdown
[[music|音楽について]]
```

### 9.2 リンクを開く

- リンクをマウスでクリックします。
- リンク上にカーソルを置いて`M-RET`を押します。
- リンク上にカーソルを置いて`C-c o f`を押します。

### 9.3 存在しないリンク先を作る

次のように、まだ存在しないノートへのリンクを書けます。

```markdown
[[new-idea]]
```

そのノートを`C-x C-s`で保存すると、`new-idea.md`が現在のノートと同じフォルダに自動作成されます。

自動作成は保存時にだけ実行されます。入力途中の文字列から大量のファイルが作成されたり、文字入力が重くなったりすることはありません。

## 10. 右パネル：グラフビュー

右パネルは、現在選択しているフォルダ内のノートとWikiリンクを図として表示します。

```text
● piano ───── ◆ music ───── ● guitar
```

- `◆ music`：現在開いているノート（拡張子は省略）
- `● piano`：別のノート（拡張子は省略）
- 線：ノート間のWikiリンク

色を表示できる環境では、現在のノートが赤、直接接続されたノートが青、孤立したノートが灰色になります。

| 操作 | 内容 |
|---|---|
| 矢印キー / `h j k l` | 仮想グラフ上のカメラを上下左右へ移動 |
| マウス左クリック | ノート名を中央パネルに開く |
| `RET` | カーソル位置のノートを開く |
| `g` | グラフを再描画 |
| `0` | 現在のノートを画面中央へ戻す |
| `<` / `>` | 右パネルの幅を変更 |

12ノード以下のグラフは右パネル内へ自動的に収まります。13ノード以上では仮想キャンバスが広がり、矢印キーでゲームのマップ操作と同じようにカメラをパンできます。配置には斥力、リンク間のバネ引力、中央への弱い重力が使われます。接続は読みにくい斜線を使わず、`─ │ ┌ ┐ └ ┘ ┼`の直交罫線で描画されます。ラベル文字を保護しつつ、線は`●`または`◆`の接続点まで届きます。

## 11. LaTeX数式

インライン数式は`$`で囲みます。

```markdown
オイラーの公式は $e^{i\pi}+1=0$ です。
```

独立した数式は`$$`で囲みます。

```markdown
$$
E = mc^2
$$
```

`C-c o t`でプレビューの表示と非表示を切り替えます。`latex`と`dvipng`が利用可能なら画像を生成します。利用できない場合は、たとえば`e^{i\\pi}`を`e⁽ⁱπ⁾`へ変換するUnicode数式プレビューを表示します。元の数式とプレビューが重複して表示されることはありません。

## 12. 設定を変更する

次を実行すると設定画面を開けます。

```text
M-x customize-group RET obsidian RET
```

主な設定項目は次の通りです。

| 変数 | 内容 |
|---|---|
| `obsidian-vault-directory` | 通常開くVault |
| `obsidian-tree-width` | 左パネルの初期幅 |
| `obsidian-graph-width` | 右パネルの初期幅 |
| `obsidian-graph-max-nodes` | グラフに表示する最大ノート数 |
| `obsidian-auto-timestamp` | 新規ノートへ日時を挿入するか |
| `obsidian-timestamp-format` | 日時の書式 |
| `obsidian-save-window-sizes` | パネル幅を次回起動用に保存するか |

設定を`init.el`へ直接書く例です。

```elisp
(setq obsidian-vault-directory "~/Documents/MyVault"
      obsidian-tree-width 28
      obsidian-graph-width 45
      obsidian-graph-max-nodes 150
      obsidian-timestamp-format "[%Y-%m-%d %H:%M]")
```

## 13. よくある問題

### `M-x obsidian`が見つからない

`load-path`に指定したパスと、実際のリポジトリの場所が一致しているか確認します。

```elisp
(add-to-list 'load-path "~/path/to/EMACS_OBSIDIAN")
(require 'obsidian)
```

設定を変更した後はEmacsを再起動してください。

### グラフに何も表示されない

- 左パネルで表示したいフォルダを選びます。
- `.md`ファイルがそのフォルダ内にあるか確認します。
- ノートを保存してから、右パネルで`g`を押します。

Vault直下を選択しているときは、異なるサブフォルダのノートを混在させないため、直下のノートだけが表示されます。

### リンクを書いてもファイルが作られない

リンクを閉じる`]]`まで入力し、`C-x C-s`で現在のノートを保存してください。

```markdown
[[作成するノート名]]
```

### LaTeXが画像にならない

ターミナルで次を確認します。

```sh
which latex
which dvipng
```

見つからない場合は、利用しているOSのパッケージ管理方法でTeX環境と`dvipng`を導入してください。導入しなくても、ギリシャ文字、上付き・下付き文字、分数、代表的な演算子をUnicodeへ変換するフォールバックを利用できます。

### 古いコードが読み込まれる

リポジトリを更新した後にEmacsを再起動します。手動で生成した古い`.elc`ファイルがある場合は削除し、もう一度起動してください。

## 14. 更新する

リポジトリのフォルダで次を実行します。

```sh
git pull
```

更新後はEmacsを再起動します。

## 15. アンインストール

1. `init.el`から`load-path`、`require`、関連する`setq`を削除します。
2. EMACS_OBSIDIANのリポジトリフォルダを削除します。

Vaultは通常のMarkdownファイル群です。パッケージを削除してもVaultを削除する必要はありません。
